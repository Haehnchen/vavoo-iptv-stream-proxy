# VAVOO.TV Streaming Proxy

This project exposes a small local proxy for Vavoo live streams through these endpoints:

- `GET /channels.m3u8`
- `GET /countries`
- `GET /stream/:id`

The proxy keeps the local interface stable and resolves the actual Vavoo stream URL when playback starts.

The proxy is optional for playback streaming. If your player can send the required `User-Agent` from the playlist, it redirects directly. Otherwise, the proxy forwards the stream.

## How It Works

- Channel lists are loaded from the current Vavoo catalog API.
- Stream URLs are resolved on demand through the current Vavoo resolve API.
- Clients can keep using the local proxy URLs generated by this project.

## Requirements

- Node.js `>=24`
- npm

## Install

```bash
npm install
```

## Start

```bash
node index.js
```

## Configuration

All configuration is done through CLI parameters.

## CLI Parameters

- `--http-host`: local HTTP host used for displayed URLs, default `127.0.0.1`
- `--http-port`: local HTTP port for the proxy, default `8888`
- `--vavoo-language`: language sent to the Vavoo APIs, default `de`, optional example: `en`
- `--vavoo-region`: region sent to the Vavoo APIs, default `US` for a broad catalog, optional `DE` which tends to prefilter strongly toward Germany
- `--vavoo-url-list`: built-in URL selection, one of `primary`, `fallback`, or `both`, default `both`

## Extended Example

```bash
node index.js \
  --http-host 127.0.0.1 \
  --http-port 8888 \
  --vavoo-language de \
  --vavoo-region US \
  --vavoo-url-list both
```

## Endpoints

### `GET /channels.m3u8`

Returns an M3U playlist.

Examples:

```text
http://<http-host>:<http-port>/channels.m3u8
http://<http-host>:<http-port>/channels.m3u8?country=Germany
http://<http-host>:<http-port>/channels.m3u8?country=Italy
http://<http-host>:<http-port>/channels.m3u8?country=France
http://<http-host>:<http-port>/channels.m3u8?country=Spain
http://<http-host>:<http-port>/channels.m3u8?country=United%20Kingdom
```

Behavior:

- Without `country`, the endpoint returns all channels.
- With `country`, the endpoint returns only matching channels.

Each entry points to the local proxy endpoint:

```text
/stream/:id
```

### `GET /countries`

Returns the available country names from the current upstream catalog.

Example:

```text
http://<http-host>:<http-port>/countries
```

### `GET /stream/:id`

Starts playback for a single channel.

Example:

```text
http://<http-host>:<http-port>/stream/18254938455ea9d58931e4
```

Behavior:

- If the client `User-Agent` contains `vavoo`, the proxy responds with a redirect to the resolved stream URL.
- For other clients, the proxy mode stays active and fetches the resolved stream and forwards it directly. This is useful for players that do not support playlist `User-Agent` piping.
